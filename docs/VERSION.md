# 📋 版本管理方案

> 版本号规则：语义化版本（Semantic Versioning）

---

## 版本号规则

```
主版本.次版本.修订号
  │      │     │
  │      │     └── 修复bug（向下兼容）
  │      └──────── 新功能（向下兼容）
  └────────── 重大变化（不兼容）
```

## 🔄 版本类型说明

| 类型 | 命名 | 说明 |
|------|------|------|
| **原型版** | 0.x | 快速迭代，不保证稳定 |
| **预发布** | x.9 | 冻结新功能，只修 bug |
| **正式版** | x.0.0 | 稳定发布 |
| **维护版** | x.x.1+ | Bug 修复 |

---

## 🏷️ 分支管理（Git Flow）

```
main (正式发布)
  │
  ├── develop (开发分支)
  │     │
  │     ├── feature/xxx (功能分支)
  │     │     │
  │     │     └── 完成后合并回 develop
  │     │
  │     └── bugfix/xxx (修复分支)
  │           │
  │           └── 完成后合并回 develop
  │
  └── release/x.x (发布分支)
        │
        └── 完成后合并到 main + 打标签
```

---

## 📦 发布流程

1. **开发** → 在 `feature/xxx` 分支开发
2. **合并** → 合并到 `develop` 分支
3. **测试** → 在 `release/x.x` 分支测试
4. **发布** → 合并到 `main` 并打标签
5. **通知** → 记录发布日志
